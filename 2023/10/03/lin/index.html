dhcpig 工具的使用
1.DHCP 的原理
工作如下：客户端启动时，通过发送 DHCP discover（动态主机配协议发送）消息广播请求 ip 地址。
DHCP 服务器收到 discover 消息，恢复 dhcp offer 消息（动态主机配置协议提供），给你提供 ip 地址，子网掩码，网关。
客户端接收到 offer 消息后，选择一个 DHCP 服务器提供 ip 地址，发送 DHCP request（动态主机配置协议请求）
DHCP 服务器收到 request 消息，发送一个 DHCP ACK 消息确认，就完成了 ip 地址分配的过程。
2.DHCP 的优点 自动化，灵活性，节省 ip 地址。
3. 安全性 1. 未经授权的 DHCP 服务器
2.ip 地址欺骗
4. 下面我介绍一个工具 dhcpig
作用：DHCP 服务负责网络的 IP 分配服务。通过攻击该服务，可以导致网络内主机获取不到 IP，而无法正常使用网络。Kali Linux 提供一款专用工具 DHCPig。该工具借助 Scapy 大量伪造 Mac 地址，而从 DHCP 服务器那里骗取 IP，消耗掉所有的 IP 资源。这样，新加入网络的计算机将无法获取 IP 地址，导致联网失败，该工具还可以借助 ARP 协议攻击局域网现有主机，造成主机离线，导致无法获取新的 IP 地址。
https://github.com/kamorin/DHCPig 在 github 里下载即可
下面是 dhcpig 的参数
- h help 显示帮助信息
- v --verbosity 显示详细信息
- 6 --ipv6 DHCPv6 默认是 DHCPv4
-1, --v6-rapid-commit 开启 RapidCommit (双向是被指定的，而不是 4 向。) (未开启)
-s, --client-src 客户端 MAC 地址列表。如 00:11:22:33:44:55,00:11:22:33:44:56 (默认：任意地址)
-O, --request-options 可选的请求编码，如 21,22,23 or 12,14-19,23 (默认: 0-80)
-f, --fuzz 任意模糊包。 (未开启)
-t, --threads 发送线程的数目。 (1)
-a, --show-arp 检测 / 打印哪个主机有 ARP 请求 (未开启)
-i, --show-icmp 检测 / 打印 ICMP 请求 (未开启)
-o, --show-options 打印 IP 地址租赁信息。(未开启)
-l, --show-lease-confirm 检测 / 打印 DHCP 回复。 (未开启)
-g, --neighbors-attack-garp 使用无故的 ARP 去碰撞网络分段。 (未开启)
-r, --neighbors-attack-release 释放所有的邻居 IP (未开启)
-n, --neighbors-scan-arp ARP 邻居扫描 (未开启)
-x, --timeout-threads 线程生成计时器。 (0.4)
-y, --timeout-dos DOS 超时 (8) (聚集 grat.arp 的等待时间)
-z, --timeout-dhcprequest DHCP 请求超时 (2)
-c, --color 开启彩色输出 (未开启)
-v, --verbosity 显示详细输出信息
